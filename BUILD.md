# Building PawVac (Android dev-client)

PawVac runs on Expo SDK 56 / RN 0.85 with native modules (SQLite now; QVAC next),
so it needs a **dev-client build** — it does **not** run in Expo Go.

## One-time recipe (verified on Android, emulator + device)

```bash
# 1. generate the native project
npx expo prebuild --platform android --no-install

# 2. apply the RN-0.85 Gradle fix (foojay 0.5.0 is incompatible with Gradle 9.x)
#    edit android/gradle/wrapper/gradle-wrapper.properties:
#      distributionUrl=…/gradle-8.13-bin.zip      # (was gradle-9.3.1-bin.zip)

# 3. build the debug APK
cd android
JAVA_HOME=<temurin-21> ./gradlew :app:assembleDebug --no-daemon -x lint

# 4. install + run (use -s <id> if multiple devices are attached)
adb -s <emulator|device> install -r app/build/outputs/apk/debug/app-debug.apk
npx expo start --dev-client        # serves the JS bundle to the installed app
```

## Notes
- `android/` is git-ignored — it's regenerated by `prebuild`. Re-apply step 2 after any regen.
- Toolchain that works here: NDK 29, Gradle 8.13, JDK 21 (Temurin), `android:required="false"`
  for optional native libs (carried over from the QVAC POC build chain).
- Expo Go is only usable on SDK ≤ the store build's version and has **no** native modules —
  fine for pure-UI review, not for this phase onward.
